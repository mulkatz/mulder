# 15 — Native Text Detection

## 1. Objective

Provide a utility module that analyzes PDF files to detect whether they contain extractable native text. Returns `has_native_text` (boolean) and `native_text_ratio` (percentage of pages with extractable text, 0–1). This data drives the Extract step's cost gate: PDFs with high native text ratio skip Document AI entirely, saving ~$10/1000 pages.

## 2. Boundaries

### In scope
- A `detectNativeText` function that accepts a PDF Buffer and returns native text detection results
- Per-page text extraction using `pdf-parse` (or equivalent)
- Calculation of `native_text_ratio` as the fraction of pages with non-trivial text content
- A configurable minimum character threshold to distinguish real text from OCR artifacts/noise
- Type definitions for the result
- Two test fixture PDFs: one with native text, one scanned/image-only
- Barrel export from core package

### Out of scope
- PDF validation (magic bytes, file size, page count limits) — that's Ingest step B4
- Uploading to Cloud Storage — that's Ingest step B4
- Creating source records in the database — that's Ingest step B4
- Document AI processing — that's Extract step B7

### Constraints
- No GCP dependencies — this is pure local PDF processing
- Must handle corrupt/unreadable PDFs gracefully (return ratio 0, not throw)
- Must not load the full PDF into memory twice — pdf-parse handles streaming
- The function is synchronous in terms of pipeline flow but async for I/O

## 3. Dependencies

### Requires (must exist)
- `packages/core/src/shared/errors.ts` — `PipelineError` class (M1-A3, 🟢)
- `packages/core/src/shared/logger.ts` — structured logging (M1-A4, 🟢)

### New dependency
- `pdf-parse` — lightweight PDF text extraction library (npm package)

### Required by (future consumers)
- B4: Ingest step (calls `detectNativeText` and writes results to source record)
- B7: Extract step (reads `native_text_ratio` to decide whether to skip Document AI)

## 4. Blueprint

### 4.1 Types — `packages/core/src/pipeline/pdf-text-detect.ts`

All types and logic in a single module (small scope, single concern):

```typescript
/** Result of native text detection on a PDF. */
export interface NativeTextResult {
  /** Whether any page has meaningful native text. */
  hasNativeText: boolean;
  /** Fraction of pages with native text (0–1). */
  nativeTextRatio: number;
  /** Total number of pages in the PDF. */
  pageCount: number;
  /** Number of pages with native text above the character threshold. */
  pagesWithText: number;
}

/** Options for native text detection. */
export interface NativeTextDetectOptions {
  /**
   * Minimum characters per page to consider it "has text".
   * Pages with fewer characters are treated as image-only.
   * Default: 50 (filters out OCR noise, page numbers, headers).
   */
  minCharsPerPage?: number;
}
```

### 4.2 Detection function — `packages/core/src/pipeline/pdf-text-detect.ts`

```typescript
/**
 * Analyzes a PDF buffer to detect native (extractable) text.
 *
 * Uses pdf-parse to extract text per page. A page is considered
 * "has text" if it contains more than `minCharsPerPage` characters
 * (default: 50). This filters out page numbers, headers, and OCR noise.
 *
 * @param pdfBuffer - The raw PDF file content
 * @param options - Detection options (character threshold)
 * @returns Native text detection result
 */
export async function detectNativeText(
  pdfBuffer: Buffer,
  options?: NativeTextDetectOptions
): Promise<NativeTextResult>
```

**Logic:**
1. Call `pdf-parse` with the buffer, requesting page-level text via the `pagerender` callback
2. For each page, count non-whitespace characters
3. A page "has text" if `chars >= minCharsPerPage` (default: 50)
4. `nativeTextRatio = pagesWithText / totalPages`
5. `hasNativeText = pagesWithText > 0`
6. If pdf-parse throws (corrupt PDF), catch the error, log a warning, and return `{ hasNativeText: false, nativeTextRatio: 0, pageCount: 0, pagesWithText: 0 }`

### 4.3 Test fixture PDFs — `fixtures/raw/`

Two minimal test PDFs:
- `fixtures/raw/native-text-sample.pdf` — A simple PDF with real embedded text (generated programmatically or from a text-to-PDF conversion). At least 3 pages.
- `fixtures/raw/scanned-sample.pdf` — A PDF with no extractable text (image-only pages). Can be a simple 1-page PDF with an embedded image.

These are generated by a helper script or inline in the test setup. Since we need real PDF files for testing, we'll create them programmatically using `pdf-lib` (a zero-dependency PDF generation library) in a test helper.

### 4.5 Package dependency

Add to `packages/core/package.json`:
```json
"pdf-parse": "^1.1.1"
```

Note: `pdf-parse` has no native dependencies and works in Node.js directly.

### 4.6 Integration

No config changes needed — the `extraction.native_text_threshold` already exists in the config schema and is used by the Extract step (B7), not by this detection module. This module just detects and returns the ratio; the threshold comparison happens in the consumer.

## 5. QA Contract

| ID | Condition | Given / When / Then |
|----|-----------|---------------------|
| QA-01 | Detects native text in text-based PDF | Given a PDF with embedded text on all pages, when `detectNativeText` is called, then `hasNativeText` is `true` and `nativeTextRatio` is `1.0`. |
| QA-02 | Returns zero ratio for image-only PDF | Given a PDF with no extractable text, when `detectNativeText` is called, then `hasNativeText` is `false` and `nativeTextRatio` is `0`. |
| QA-03 | Calculates correct ratio for mixed PDF | Given a PDF where 2 of 4 pages have text, when `detectNativeText` is called, then `nativeTextRatio` is `0.5`. |
| QA-04 | Returns correct page count | Given a 3-page PDF, when `detectNativeText` is called, then `pageCount` is `3`. |
| QA-05 | Respects minCharsPerPage threshold | Given a PDF with a page containing only 10 characters, when `detectNativeText` is called with default threshold (50), then that page is not counted. When called with `minCharsPerPage: 5`, that page is counted. |
| QA-06 | Handles corrupt PDF gracefully | Given an invalid/corrupt buffer, when `detectNativeText` is called, then it returns `{ hasNativeText: false, nativeTextRatio: 0, pageCount: 0, pagesWithText: 0 }` without throwing. |
| QA-07 | Handles empty buffer gracefully | Given an empty Buffer, when `detectNativeText` is called, then it returns gracefully without throwing. |
| QA-08 | Build compiles without errors | When `pnpm turbo run build` is run, then the build succeeds with zero TypeScript errors. |
| QA-09 | Biome lint passes | When `npx biome check .` is run, then no lint or format violations are found. |
